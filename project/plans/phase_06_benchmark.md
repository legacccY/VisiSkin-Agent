# 阶段六：评测基准 ITB（Iterative Triage Benchmark）

## 目标

构建包含 4 个子集的标准化评测基准，对整个系统进行系统性评估，输出可直接用于论文的实验对比数据和可视化图表。

---

## 前置条件

- 阶段五完成：完整 Agent 系统可运行
- `agent/eval_agent.py` 已实现基础评测逻辑

---

## 本阶段工具

| 工具 | 用途 |
|------|------|
| scipy | t-test 显著性检验、置信区间计算 |
| scikit-learn | AUC、F1、混淆矩阵等分类指标 |
| matplotlib + seaborn | 论文用图（对比柱状图、分布图） |
| pandas | 汇总 itb_results.csv，方便筛选和统计 |
| wandb | 多 run 对比 dashboard，实验结果一目了然 |

---

## 关键技术决策点

1. **每个子集样本数**：100 / 500 / 1000。推荐 500（统计显著性 vs 运行时间的折中）

2. **对比 baseline 选择**（选 2-3 个）：
   - 直接诊断（无质量过滤）
   - BRISQUE 质量过滤 + 传统分类器
   - 无 Agent 交互的端到端模型（VisiScore + QAD，不追问用户）

3. **人工评估是否参与**：全自动 / 加入小规模人工评估（5-10 人）用于验证 Agent 交互质量

---

## 交付物清单

| 文件 | 用途 |
|------|------|
| `benchmark/build_itb.py` | 构建 4 个子集，从数据集中采样并标注 |
| `benchmark/metrics.py` | 评估指标计算：分诊准确率/交互轮次/遵从率/质量改善率 |
| `run_experiments.py` | 批量实验脚本：对所有 baseline 和我们的系统跑完整评测 |
| `analyze_results.py` | 结果统计分析 + matplotlib/seaborn 可视化 |
| `results/itb_results.csv` | 所有实验的原始数字 |
| `results/figures/` | 论文用图：对比柱状图、交互轮次分布图、质量改善示例图 |

---

## ITB 4 个子集定义

| 子集 | 内容 | 评测目的 |
|------|------|---------|
| **ITB-LQ**（低质量） | 人工退化的低质量皮肤图 | 测试系统识别低质量并引导追问的能力 |
| **ITB-HQ**（高质量） | 原始高质量皮肤图 | 测试系统在好图上的诊断准确率 |
| **ITB-Edge**（边界案例） | 质量评分在阈值附近的模糊案例 | 测试系统在不确定情况下的鲁棒性 |
| **ITB-Diverse**（多样性） | 多肤色（FitzPatrick I-VI）多病灶类型 | 测试系统的公平性和泛化性 |

---

## 工作步骤

1. **构建 4 子集**：从现有数据集中按子集标准采样，确保各子集互不重叠
2. **定义评估指标**：确认指标计算逻辑，写 `metrics.py` 并单元测试
3. **运行全量实验**：对所有 baseline 和本系统跑 ITB，记录原始结果
4. **统计分析**：置信区间、显著性检验（t-test），确保结论统计可靠
5. **可视化出图**：生成论文质量的图表（字体/颜色/尺寸符合 MICCAI 要求）

---

## 验收标准

- 4 个子集全部构建完成，总样本量达到设计规模
- 本系统在 ITB-LQ 子集上显著优于所有 baseline（这是论文核心实验）
- `analyze_results.py` 输出的图表可直接放入论文（无需额外美化）

---

## 硬件/资源约束

- 批量实验可能需要几小时 GPU 时间（每个 baseline × 4 子集 × 样本数）
- 建议使用 `run_experiments.py` 的 `--baseline` 参数逐个运行，方便断点恢复
- 统计分析（CPU only）无显存压力

---

## 注意事项

- ITB-Diverse 子集需要覆盖 Fitzpatrick I-VI 各肤色，注意采样均衡
- 人工遵从率（用户按照 Agent 建议重拍的比率）如果没有真实用户，可用模拟：Agent 追问后自动提供"改善版"图片代替用户重拍
- 所有随机采样固定 seed，保证实验可复现
- 图表要提前确认 MICCAI 2027 的页面宽度和 DPI 要求，避免论文写作阶段返工
